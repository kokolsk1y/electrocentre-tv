## Фаза 1: Стабилизация и полировка

**Цель:** Устранить критические дефекты и визуальные недостатки v2.1 так, чтобы файл был готов к демонстрации руководству.

**Покрывает требования:** SLIDE-01, SLIDE-02, SLIDE-03, SLIDE-04, CONTENT-01, CONTENT-02, CONTENT-03, CONTENT-04, CONTENT-05, TECH-01, TECH-02, TECH-03, TECH-04, TECH-05

**Условие входа:** Файл `electrocentre-tv-v2.1.html` существует и открывается в браузере. Основной слайдер работает.

**Условие выхода:**
- Файл открывается на Android TV по ссылке GitHub Pages без ошибок в консоли
- Переход между слайдами плавный (cross-fade 400мс), белая вспышка отсутствует
- Шрифт Montserrat отображается корректно без подключения к интернету (self-hosted или base64)
- Имена файлов баннеров переведены в ASCII-формат (`banner-01.png` и т.д.), ссылки в HTML обновлены
- Слайд 7 (доставка) содержит оба варианта условий, текст читается с расстояния 2–3 метров
- Анимация на HTML-слайдах работает без сбоев при первом и повторном проходе цикла
- Разработчик отправил скриншоты/запись экрана руководству для предварительного согласования

**Сложность:** Средняя

**Ключевые риски:**
- Self-hosted шрифт увеличит размер файла на ~200–400 KB — нужно проверить скорость загрузки на TV
- Переход на cross-fade требует изменения механизма слайдера — риск сломать анимацию элементов (SLIDE-04)
- Переименование баннеров требует синхронного обновления HTML, иначе слайды станут пустыми
- Неизвестна точная модель TV: если браузер не поддерживает `woff2` — шрифт не загрузится

**Задачи:**
1. Встроить шрифт Montserrat (woff2, начертания 300/400/600/700/800/900) как `@font-face` в `<style>`
2. Переименовать файлы баннеров в `banner-01.png … banner-05.png`, обновить `src` в HTML
3. Заменить механизм flash-overlay на cross-fade через CSS `opacity` и классы `.leaving` / `.entering`
4. Улучшить слайд 7 (CONTENT-04): добавить оба варианта доставки, укрупнить условия
5. Проверить анимацию HTML-слайдов после переделки слайдера
6. Открыть на TV, снять скриншоты/видео, отправить руководству

**Критерии успеха (автоматически проверяемые):**
- Файл HTML валиден и открывается без ошибок в браузере Chrome/Chromium
- В коде отсутствуют строки `fonts.googleapis.com` (шрифт встроен)
- В атрибутах `src` отсутствует кириллица (проверить поиском по файлу)

**Критерии успеха (ручная проверка):**
- На TV отсутствует белая вспышка при смене слайдов
- Шрифт отображается корректно при отключённом Wi-Fi
- Слайд 7 читается с 2 метров без напряжения

## Фаза 2: Одобрение и самостоятельное управление контентом

**Цель:** Получить официальное одобрение руководства и дать команде магазина возможность обновлять баннеры без разработчика.

**Покрывает требования:** APPROVE-01, MGMT-01, MGMT-02

**Условие входа:** Фаза 1 завершена. Руководство (маркетолог + директор + другие) видело демонстрацию и готово к финальному собранию.

**Условие выхода:**
- APPROVE-01: руководство официально одобрило систему на собрании, зафиксированы финальные правки
- MGMT-01 + MGMT-02: менеджер без технического опыта может заменить баннер за 15 минут по письменной инструкции
- Инструкция по обновлению баннеров передана команде магазина

**Сложность:** Средняя

**Ключевые риски:**
- Руководство может запросить значительные изменения контента после одобрения — нужен буфер времени для правок
- Архитектура обновления баннеров (manifest.json vs Google Sheets vs ручная загрузка на GitHub) зависит от модели TV и поддержки `fetch()` — необходимо проверить на реальном устройстве до выбора подхода
- Если `fetch()` не работает на TV — самостоятельное обновление без разработчика технически невозможно в v2; в этом случае процесс обновления остаётся через разработчика (5 минут по заданию из PROJECT.md)
- Нетехнический сотрудник может случайно сломать JSON/конфиг — нужна валидация или очень простой интерфейс

**Задачи:**
1. Провести финальный показ руководству (разработчик + команда магазина)
2. Зафиксировать правки после собрания и внести их в файл
3. Исследовать поддержку `fetch()` на реальном TV
4. По результату теста выбрать механизм обновления: manifest.json (если fetch работает) или ручная замена файлов + коммит на GitHub
5. Написать инструкцию по обновлению баннеров для нетехнического сотрудника
6. Провести тест: менеджер магазина самостоятельно меняет баннер по инструкции, хронометраж <= 15 минут

**Критерии успеха (автоматически проверяемые):**
- Если выбран manifest.json: `manifest.json` валидный JSON, TV подхватывает новый баннер в течение следующего цикла обновления
- Инструкция существует как отдельный документ и передана команде

**Критерии успеха (ручная проверка):**
- Руководство подтвердило одобрение (устно или письменно)
- Нетехнический сотрудник сменил баннер за <= 15 минут без помощи разработчика

## Фаза 3: Расширение функциональности

**Цель:** Добавить защиту от выгорания экрана, живую дату в шапку и навигационный видеоролик после того, как система стабильно работает и одобрена.

**Покрывает требования:** SLIDE-05, SLIDE-06, CONTENT-06

**Условие входа:** Фаза 2 завершена. Система работает в магазине без замечаний не менее двух недель. Навигационный ролик снят и передан разработчику.

**Условие выхода:**
- Скринсейвер активируется в нерабочие часы (настраиваемое время начала/конца) и защищает матрицу
- Дата отображается в шапке рядом с часами и обновляется ежесуточно
- Навигационный ролик встроен как отдельный слайд и корректно воспроизводится на TV

**Сложность:** Низкая (SLIDE-06) / Средняя (SLIDE-05, CONTENT-06)

**Ключевые риски:**
- CONTENT-06: видеофайл может быть тяжёлым (>20 MB) — нужно согласовать формат и сжатие до съёмки; Smart TV браузер может не поддерживать `autoplay` для видео-элементов
- SLIDE-05: скринсейвер должен корректно возобновлять слайдер при открытии магазина, не ломая цикл
- SLIDE-06: минимальный риск — 3 строки JS, но нужно проверить локаль отображения даты на конкретном TV

**Задачи:**
1. Добавить отображение даты в шапку (рядом с часами, та же функция `updateClock`)
2. Реализовать скринсейвер: проверка `new Date().getHours()` → вне рабочих часов → показывать анимированный логотип, скрыть слайды
3. Получить навигационный ролик от команды магазина, конвертировать в совместимый формат (mp4/h264)
4. Встроить видеослайд в цикл, проверить на TV (autoplay muted)

**Критерии успеха (автоматически проверяемые):**
- В коде присутствует логика скринсейвера с проверкой времени
- Дата обновляется в DOM без перезагрузки страницы

**Критерии успеха (ручная проверка):**
- В нерабочие часы TV показывает движущийся логотип, а не зафиксированный слайд
- Навигационный ролик воспроизводится на TV без звука (muted) и без кнопок управления
